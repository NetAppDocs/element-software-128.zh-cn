= 更换 H410S 节点
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
如果存储节点出现 CPU 故障、Radian 卡问题、其他主板问题或无法开机等情况，则应更换存储节点。这些说明适用于 H410S 存储节点。

NetApp Element软件用户界面中的警报会在存储节点发生故障时发出提醒。您应该使用 Element UI 获取故障节点的序列号（服务标签）。您需要这些信息来定位集群中发生故障的节点。

这是一台 2U 四节点机箱的背面，机箱内有四个存储节点：

image::hci_stornode_rear.gif[该图显示的是具有四个存储节点的四节点机箱。]

这是配备 H410S 节点的四节点机箱的正面视图，图中显示了与每个节点对应的插槽：

image::hci_stor_node_ssd_bays.gif[显示与四节点机箱中每个节点关联的插槽，该机箱包含 H410S 节点。]

.你需要什么
* 您已确认您的存储节点出现故障，需要更换。
* 您已获得替换存储节点。
* 您佩戴了静电放电 (ESD) 腕带，或者采取了其他防静电防护措施。
* 您已为连接到存储节点的每根电缆贴上了标签。


以下是各步骤的概要：

* <<准备替换节点>>
* <<更换机箱内的节点。>>
* <<将节点添加到集群>>




== 准备替换节点

在安装替换节点之前，您应该在NetApp Element软件 UI 中正确地从集群中删除故障存储节点。这样做不会造成任何服务中断。您应该从 Element UI 获取故障存储节点的序列号，并将其与节点背面标签上的序列号进行匹配。

.步骤
. 在 Element UI 中，选择“集群”>“驱动器”。
. 使用以下方法之一从节点中移除驱动器：
+
[cols="2*"]
|===
| 选项 | 步骤 


 a| 
移除单个驱动器
 a| 
.. 点击要删除的驱动器旁边的“操作”。
.. 单击“*删除*”。




 a| 
移除多个驱动器
 a| 
.. 选择要删除的所有驱动器，然后单击“批量操作”。
.. 单击“*删除*”。


|===
. 选择“集群”>“节点”。
. 记下故障节点的序列号（服务标签）。你应该将它与节点背面标签上的序列号进行匹配。
. 记下序列号后，按如下方式从集群中移除节点：
+
.. 选择要删除的节点的“操作”按钮。
.. 选择*删除*。






== 更换机箱内的节点。

使用NetApp Element软件 UI 从集群中移除故障节点后，即可将该节点从机箱中物理移除。您应该将替换节点安装到您从中移除故障节点的机箱的同一插槽中。

.步骤
. 操作前请佩戴防静电装备。
. 打开新存储节点的包装，将其放置在机箱附近的水平面上。
+
请保留包装材料，以便在将故障节点退回NetApp时使用。

. 将要移除的存储节点背面插入的每根电缆贴上标签。
+
安装好新的存储节点后，应该将电缆插入原来的端口。

+
下图显示的是存储节点的背面：

+
image::../media/hci_isi_storage_cabling.png[该图显示了存储节点的布线情况。]

+
[cols="2*"]
|===
| 端口 | 详细信息 


 a| 
A 港
 a| 
1/10GbE RJ45 端口



 a| 
B 港
 a| 
1/10GbE RJ45 端口



 a| 
C端口
 a| 
10/25GbE SFP+ 或 SFP28 端口



 a| 
D港
 a| 
10/25GbE SFP+ 或 SFP28 端口



 a| 
IPMI
 a| 
1/10GbE RJ45 端口

|===
. 断开存储节点上的所有电缆。
. 拉下节点右侧的凸轮手柄，然后用两个凸轮手柄将节点拉出。
+
向下拉动的凸轮手柄上有一个箭头，指示其移动方向。另一个凸轮手柄不会移动，它的作用是帮助你将节点拉出。

+
image::../media/hci_stor_node_camhandles.gif[图中]

+
[cols="2*"]
|===
| 物品 | 描述 


 a| 
1
 a| 
凸轮手柄可帮助您将节点拉出。



 a| 
2
 a| 
在拔出节点之前，需要向下扳动凸轮手柄。

|===
+

NOTE: 将节点从机箱中拉出时，请用双手托住节点。

. 将节点放置在水平面上。
+
您必须将节点打包并返回给NetApp。

. 将替换节点安装到机箱的同一插槽中。
+

IMPORTANT: 将节点滑入机箱时，请勿用力过猛。

. 将已移除节点上的硬盘驱动器移出，并插入新节点中。
. 将电缆重新连接到您之前断开连接的端口。
+
你断开电缆时电缆上的标签会对你有所帮助。

+
[NOTE]
====
.. 如果机箱后部的通风口被电缆或标签堵塞，可能会因过热而导致组件过早损坏。
.. 请勿强行将线缆插入端口；否则可能会损坏线缆、端口或两者都损坏。


====
+

TIP: 确保替换节点的布线方式与机箱内其他节点的布线方式相同。

. 按下节点正面的按钮即可开机。




== 将节点添加到集群

当您向集群添加节点或在现有节点中安装新驱动器时，这些驱动器会自动注册为可用。必须先使用 Element UI 或 API 将驱动器添加到集群中，然后它们才能参与集群。

集群中每个节点上的软件版本应该兼容。当您向集群添加节点时，集群会根据需要在新节点上安装集群版本的 Element 软件。

.步骤
. 选择“集群”>“节点”。
. 选择“待处理”以查看待处理节点列表。
. 执行以下操作之一：
+
** 要添加单个节点，请选择要添加的节点的“操作”图标。
** 要添加多个节点，请选中要添加的节点的复选框，然后选择“批量操作”。
+

NOTE: 如果您添加的节点的 Element 软件版本与集群上运行的版本不同，则集群会异步地将该节点更新为集群主节点上运行的 Element 软件版本。节点更新后，会自动添加到集群中。在此异步过程中，节点将处于 `pendingActive`状态。



. 选择“*添加*”。
+
该节点出现在活动节点列表中。

. 从 Element UI 中选择“集群”>“驱动器”。
. 选择“可用”以查看可用驱动器列表。
. 执行以下操作之一：
+
** 要添加单个驱动器，请选择要添加的驱动器的“操作”图标，然后选择“添加”。
** 要添加多个驱动器，请选中要添加的驱动器的复选框，选择“批量操作”，然后选择“添加”。
