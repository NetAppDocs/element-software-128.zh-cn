= 获取集群版本信息
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
你可以使用 `GetClusterVersionInfo`检索集群中每个节点上运行的 Element 软件版本信息的方法。该方法还会返回有关当前正在升级软件的节点的信息。



== 集群版本信息对象成员

该方法具有以下对象成员：

|===
| 名称 | 描述 | 类型 


 a| 
nodeID
 a| 
节点的 ID。
 a| 
整数



 a| 
节点内部修订
 a| 
节点的内部软件版本。
 a| 
string



 a| 
节点版本
 a| 
节点的软件版本。
 a| 
string

|===


== 参数

此方法没有输入参数。



== 返回值

此方法具有以下返回值：

|===
| 名称 | 描述 | 类型 


 a| 
集群 API 版本
 a| 
集群上当前的 API 版本。
 a| 
string



 a| 
集群版本
 a| 
集群上当前运行的 Element 软件版本。
 a| 
string



 a| 
集群版本信息
 a| 
集群中节点的列表，以及每个节点的版本信息。
 a| 
JSON 对象数组



 a| 
待定集群版本
 a| 
如果存在，则表示集群软件当前正在升级或回滚到的版本。
 a| 
string



 a| 
软件版本信息
 a| 
升级状态。对象成员：

* 当前版本：
+
节点上当前的软件版本。

* nodeID：要从 currentVersion 升级到 pendingVersion 的节点的 ID。如果没有正在进行的升级，则此字段为 0（零）。
* packageName：要安装的软件包的名称。
* pendingVersion：正在安装的软件版本。
* startTime：安装开始的日期和时间，采用 UTC+0 格式。

 a| 
JSON 对象

|===


== 请求示例

该方法的请求类似于以下示例：

[listing]
----
{
   "method": "GetClusterVersionInfo",
   "params": {},
   "id" : 1
}
----
