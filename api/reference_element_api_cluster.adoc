= 集群
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
集群对象包含节点用于与集群通信的信息。您可以使用 GetClusterConfig API 方法检索此信息。



== 对象成员

该对象包含以下成员：

|===
| 名称 | 描述 | 类型 


 a| 
cipi
 a| 
用于集群通信的网络接口。
 a| 
string



 a| 
集群
 a| 
唯一的集群名称。
 a| 
string



 a| 
具备加密能力
 a| 
指示节点是否支持驱动器加密。
 a| 
布尔值



 a| 
合奏
 a| 
参与集群的节点。
 a| 
字符串数组



 a| 
fipsDriveConfiguration
 a| 
指示节点是否支持 FIPS 140-2 认证的驱动器。
 a| 
布尔值



 a| 
mipi
 a| 
用于节点管理的网络接口。
 a| 
string



 a| 
name
 a| 
集群名称。
 a| 
string



 a| 
nodeID
 a| 
集群中节点的节点 ID。
 a| 
string



 a| 
待处理节点ID
 a| 
集群中待处理节点的 ID。
 a| 
整数



 a| 
角色
 a| 
确定节点的角色。
 a| 
整数



 a| 
sipi
 a| 
用于存储流量的网络接口。
 a| 
string



 a| 
状态
 a| 
节点的当前状态。可能值：

* 可用：该节点尚未配置集群名称。
* 待添加：该节点正在等待加入指定的集群，可以添加。
* 活动节点：该节点是集群的活动成员，不能添加到其他集群。
* 待激活：该节点目前正在恢复到出厂软件映像，尚未成为集群的活动成员。完成后，它将变为活动状态。

 a| 
string



 a| 
version
 a| 
节点上运行的软件版本。
 a| 
string

|===
