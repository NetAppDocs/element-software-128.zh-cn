= drive
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
驱动器对象包含有关集群活动节点中各个驱动器的信息。此对象包含有关已添加为卷元数据或块驱动器的驱动器的详细信息，以及尚未添加但可用的驱动器的详细信息。您可以使用以下方式检索此信息： `ListDrives` API 方法。



== 对象成员

该对象包含以下成员：

|===
| 名称 | 描述 | 类型 


 a| 
属性
 a| 
JSON 对象格式的名称-值对列表。该对象始终为空且不可修改。
 a| 
JSON 对象



 a| 
容量
 a| 
驱动器的总容量，以字节为单位。
 a| 
整数



 a| 
底盘插槽
 a| 
对于 HCI 平台，此值为服务器机箱中该驱动器所在的节点字母和插槽号。对于存储平台而言，插槽号是“插槽”整数的字符串表示形式。
 a| 
string



 a| 
驱动器故障详情
 a| 
如果驱动器的状态为“失败”，则此字段提供有关驱动器被标记为失败原因的更多详细信息。
 a| 
string



 a| 
驱动器ID
 a| 
此驱动器的 ID。
 a| 
整数



 a| 
驱动安全故障原因
 a| 
如果启用或禁用驱动器安全功能失败，请说明失败原因。如果值为“none”，则表示没有发生故障。
 a| 
string



 a| 
密钥ID
 a| 
密钥提供商用于获取解锁此驱动器的身份验证密钥的密钥 ID。
 a| 
UUID



 a| 
密钥提供程序ID
 a| 
确定用于解锁此驱动器的身份验证密钥的提供者。
 a| 
整数



 a| 
nodeID
 a| 
包含此驱动器的节点的 ID。
 a| 
整数



 a| 
分段文件大小
 a| 
驱动器的段文件大小，以字节为单位。
 a| 
整数



 a| 
序列号
 a| 
硬盘序列号。
 a| 
string



 a| 
slot
 a| 
服务器机箱中该驱动器所在的插槽编号，如果使用 SATADimm 设备作为内部元数据驱动器，则为 -1。
 a| 
整数



 a| 
状态
 a| 
驱动器的状态。可能值：

* 可用：一个可用的驱动器。
* 活动：一个正在活动的驱动器。
* 擦除：驱动器正在被安全擦除。该硬盘上的所有数据都将被永久删除。
* 失败：硬盘发生故障。之前存储在该驱动器上的所有数据都已迁移到集群中的其他驱动器。
* 移除：正在移除一个驱动器。驱动器上先前的所有数据都将迁移到集群中的其他驱动器。

 a| 
string



 a| 
type
 a| 
驱动类型。可能值：

* 卷：存储卷元数据。
* 块：存储块数据。
* 未知：驱动器类型尚未激活，尚待确定。

 a| 
string



 a| 
可用容量
 a| 
驱动器的可用容量，以字节为单位。
 a| 
整数

|===
