= 配置SNMP陷阱
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
系统管理员可以使用 SNMP 陷阱（也称为通知）来监控SolidFire集群的运行状况。

启用 SNMP 陷阱后， SolidFire集群会生成与事件日志条目和系统警报关联的陷阱。要接收 SNMP 通知，您需要选择要生成的陷阱并确定陷阱信息的接收者。默认情况下，不会生成任何陷阱。

. 点击“集群”>“SNMP”。
. 在“SNMP陷阱设置”部分，选择系统应生成的一种或多种陷阱类型：
+
** 簇故障陷阱
** 集群已解决的故障陷阱
** 集群事件陷阱


. 在“*Trap Recipients*”部分，输入接收者的主机、端口和团体字符串信息。
. *可选*：要添加另一个陷阱接收者，请单击*添加陷阱接收者*并输入主机、端口和团体字符串信息。
. 点击“保存更改”。
