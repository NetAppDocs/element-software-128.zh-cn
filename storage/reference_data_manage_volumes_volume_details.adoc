= 容量详情
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
“管理”选项卡上的“卷”页面提供有关活动卷的信息，例如名称、帐户、与卷关联的访问组以及卷的大小。

* *ID*
+
系统生成的卷 ID。

* *姓名*
+
该卷创建时所起的名称。

* *帐户*
+
分配给该卷的帐户名称。

* *访问组*
+
卷所属的卷访问组的名称。

* *使用权*
+
创建卷时分配给该卷的访问类型。可能值：

+
** 读/写：所有读写操作均被接受。
** 只读：允许所有读取操作；不允许写入操作。
** 已锁定：仅允许管理员访问。
** ReplicationTarget：指定为复制卷对中的目标卷。


* *用过的*
+
体积中已使用空间的百分比。

* *尺寸*
+
卷的总大小（以 GB 为单位）。

* *快照*
+
为该卷创建的快照数量。

* *服务质量策略*
+
用户自定义 QoS 策略的名称和链接。

* *最低IOPS*
+
该卷保证的最低 IOPS 数。

* *最大IOPS*
+
该卷允许的最大 IOPS 数。

* *突发IOPS*
+
在短时间内允许该卷的最大 IOPS 数。默认值 = 15,000。

* *属性*
+
已通过 API 方法以键/值对的形式分配给卷的属性。

* *512e*
+
指示卷上是否启用了 512e。可能值：

+
** 是
** 否


* 创建于*
+
卷册的创建日期和时间。
